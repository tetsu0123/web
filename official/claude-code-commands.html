🚀 基本コマンド＆オプション

📝 基本的な使用方法

# 対話モードで起動 claude # ヘッドレスモード（非対話）でプロンプト実行 claude -p "プロンプトをここに入力" # 出力形式を指定（text, json, stream-json） claude -p "バグを修正して" --output-format json # 詳細ログ表示（デバッグ用） claude -p "テストを実行" --verbose # 権限チェックを無効化（危険） claude --dangerously-skip-permissions # 許可するツールを指定 claude --allowedTools Bash,Edit,Read,Write

⚡ 実行モード切り替え

🔄 ノーマルモード

デフォルト状態。各操作で確認プロンプトが表示されます。

切り替え: 対話モード中に Shift + Tab

🔥 自動実行モード

権限プロンプトをスキップして実行します。

起動時指定: --dangerously-skip-permissions

📋 プランモード

実行前に詳細な計画を表示。戦略的な作業に最適。

切り替え: Shift + Tab でモード切り替え

⚠️ --dangerously-skip-permissions 使用時の注意

このフラグはすべての権限チェックを無効化し、Claudeがファイル編集、コマンド実行、システム操作を確認なしで実行します。Anthropic公式では「インターネット接続のないDockerコンテナ内でのみ使用」を推奨していますが、十分注意して使用してください。

⚡ --dangerously-skip-permissions の具体的使用例

# 基本的な使用方法 claude --dangerously-skip-permissions # プロンプト指定での非対話実行 claude --dangerously-skip-permissions -p "リントエラーを全て修正して" # プロジェクト全体の自動セットアップ claude --dangerously-skip-permissions -p "Next.js + TypeScript + Tailwindでプロジェクト作成" # 大量ファイルの一括変換 claude --dangerously-skip-permissions -p "srcフォルダ内の全.jsファイルを.tsに変換" # 自動テスト修正 claude --dangerously-skip-permissions -p "失敗しているテストを全て修正" # エイリアス作成（よく使う場合） alias cc="claude --dangerously-skip-permissions" cc -p "バグを修正してコミット"

🛡️ セキュリティ考慮事項

  • --dangerously-skip-permissions は慎重に使用（公式推奨：Dockerコンテナ内のみ）
  • allowedTools設定で実行可能な操作を制限
  • 重要なファイルのバックアップを事前に作成
  • git worktreeを使って並列Claude セッションを安全に実行
  • --max-turnsで実行回数を制限
  • --verboseでデバッグ時の動作を詳細に記録
